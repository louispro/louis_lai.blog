Java

一、查漏补缺

  • 在java中构造器不能定义返回值类型，也不能使用void声明构造器没有返回值。如果为构造器定义了返回值类型，或使用void声明构造器没有返回值，编译时不会出错，但java会把这个所谓的构造器当成方法来处理，也就是说它不再是 构造器了。
  • 在java中方法不能独立存在，必须属于类或对象

二、static

2.1、static修饰方法

在《Java编程思想》是这样介绍static关键字的：

`static方法就是没有this的方法。在static方法内部不能调用非静态方法，反过来是可以的。而且可以在没有创建任何对象的前提下，仅仅通过类本身来调用static方法。这实际上正是static方法的主要用途。

静态成员为什么不能访问非静态成员？

换个简单的说法，创建一个类，类里分别有一个类变量，类方法（类方法中访问了实例变量，虽然这是错的）和实例变量，在没有创建实例对象的情况下，直接用类名调用类方法，此时对象不存在，实例变量还不存在，也就无法在类方法中访问一个不存在的实例变量了，所以java设计为静态成员不能访问非静态成员。

2.2、static修饰变量

​ static变量也称作静态变量，静态变量和非静态变量的区别是：静态变量被所有的对象所共享，在内存中只有一个副本，它当且仅当在类初次加载时会被初始化。而非静态变量是对象所拥有的，在创建对象的时候被初始化，存在多个副本，各个对象拥有的副本互不影响。static成员变量的初始化顺序按照定义的顺序进行初始化。

2.3、static修饰代码块

​ static关键字还有一个比较关键的作用是用来形成静态代码块以优化程序性能。static可以置于类中的任何地方，类中可有有多个static块。再类初次加载的时候，会按照static块的顺序来执行每个static块，并且只能执行一次。系统将在类初始化阶段执行静态初始化块，而不是再创建对象时才执行。因此静态初始化块总是比普通初始化先执行。系统再在类的初始化阶段执行静态代码块时，不仅会执行本类的静态初始化块，而且还会一直追溯到java.lang.Obeject类，先执行java.lang.Obeject的静态初始化块，然后执行器父类的静态初始化块，最后才执行该类的静态初始化块，经过这个过程，才完成了该类的初始化过程。同理，静态初始化块也不能访问非静态成员。

2.4、static修饰内部类

​ 使用static修饰的内部类属于外部类本身，而不是外部类的某个对象（外部类不能使用static修饰）。静态内部类作为一个类其内部可以包含静态成员和非静态成员，但其作为外部类的静态成员则需要遵循静态成员不能访问非静态成员的规则，所以静态内部类不能访问外部类的实例成员，只能访问外部类的的静态成员。与普通内部类不同的是，外部类不能直接访问普通内部类的成员变量，但是外部类可以用静态内部类名访问静态内部类的类成员。
